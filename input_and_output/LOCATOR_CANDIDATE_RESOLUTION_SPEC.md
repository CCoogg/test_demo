# LOCATOR_CANDIDATE_RESOLUTION_SPEC.md

## Vision 点击 → 稳定 Locator 生成策略

------------------------------------------------------------------------

# 一、文档目标

解决问题：

AI 通过截图识别并点击元素 → 产生坐标\
如何将"坐标点击"转化为"可稳定固化的自动化 Locator"？

目标：

-   不直接使用坐标
-   不盲目只保存 resourceId
-   生成稳定、可裁决、可复用的 locator_candidate
-   为 Registry 构建提供可操作数据

------------------------------------------------------------------------

# 二、核心原则

1.  坐标不是 Locator，只是操作证据。
2.  不反推单一元素，而是构造候选集合。
3.  Agent 输出"证据"即候选集合，下游Framework 负责"裁决"。
4.  优先构造稳定策略组合，而非单字段定位。

------------------------------------------------------------------------

# 三、操作

-   点击前，获取当前 UI 树（uiautomator dump）
-   只提取 bounds包含点击点的subtree

------------------------------------------------------------------------

# 四、候选元素集合构造规则

从点击前 dump 中筛选：

1.  所有 bounds 包含 (x, y) 的节点
2.  若无匹配 → 允许扩大 3\~5px 容差

------------------------------------------------------------------------

# 五、候选裁决优先级算法

优先使用：

resourceId > content-desc > text > class + hierarchy

最终选择 Top 1 作为 primary candidate\
其余保留为 fallback

------------------------------------------------------------------------

# 六、Locator Candidate 输出结构（必须遵循）

示例：

{ "locator_candidate": { "strategies": \[ {"by": "id", "value":
"com.xxx:id/login_btn"}, {"by": "text", "value": "登录"}, {"by":
"class_chain", "value": "android.widget.Button\[1\]"} \], "confidence":
0.82 } }

规则：

-   不只输出一个字段
-   strategies 按优先级排序
-   confidence 根据裁决稳定性计算

------------------------------------------------------------------------

# 七、confidence 计算建议

基础分：

-   有 resourceId：+0.4
-   clickable=true：+0.2
-   有 text：+0.2

总分 ≥ 0.8 → 高可信\
0.6\~0.8 → 需人工确认\
\<0.6 → 不建议固化

------------------------------------------------------------------------

# 八、特殊场景处理

## 容器被点击

若节点：

-   无 resourceId
-   无 text
-   clickable=false

则：

-   查找最近 clickable 子节点
-   若无 → 查找最近 clickable 父节点

------------------------------------------------------------------------

# 九、Registry 构建建议

Agent 不直接写入 Registry。

流程应为：

1.  Agent 输出 locator_candidate，即本项目的目标，后续内容仅做提示，与本项目无关
2.  Framework 裁决
3.  人工审核（初期）
4.  写入 LOCATOR_REGISTRY

------------------------------------------------------------------------

# 十、最终目标

将 Vision 不稳定点击：

(x, y)

转化为：

稳定、可维护、可复用的 DSL locator_key

------------------------------------------------------------------------
